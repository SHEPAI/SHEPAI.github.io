Introduction


SHEP AI

Advancing the evolution

SHEP has been in development for many years. SHEP started off a keyboard input Arduino board, using an SD card and text file to store data. SHEP moved on to be voice controlled on the Arduino, using a series of boards communicating with eachother as a network. Later on, SHEP became a python program on the Raspberry Pi. Different inputs were used with this program from audio to network cables.

A python library for SHEP was made so SHEP could be used to control any system, with ease. Different products are being developed using this new method. From version 0.0.6 we have SHEP using the internet to enhance it's knowledge, without the user telling it.

The AI matrix is a system which collects an input, checks its data and outputs the found string of words to the user. If the input is not found in the data, the AI will ask the user to add it. The input will then be in the solid state memory and accessible in the future when the AI is faced with this input. The output can be from saying words to physically learning through potentiometers and servos. SHEP has a network of files where an input is checked word for word to find an appropriete reply to the user.

This all sounds rather technical, to sum it up; SHEP is a learning system which gets smarter the more you talk to it and teach it. We hope to see SHEP controlling major systems such as: Aircraft, battleships, large buildings, maybe even cities. SHEP is open source which means SHEP can be adapted to any application.

SHEP has been in development for 5 years now, from a small arduino board to a series of networks. SHEP is still experimental and is being adapted constantly. We ask that any distribution of SHEP's software leave credit to the origional developers.
